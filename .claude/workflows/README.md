## ワークフローの使い方

### 手動実行

1. ワークフローファイルを開く
2. 各エージェントのタスクを確認
3. `send-message.sh` でタスクを配布

```bash
# 例: three-js-visualization ワークフローを実行
./send-message.sh エージェント1 "タスク1の内容"
./send-message.sh エージェント2 "タスク2の内容"
./send-message.sh エージェント3 "タスク3の内容"
```

### 自動実行（将来対応予定）

```bash
# orchestrate.sh を使った自動実行（実装中）
./orchestrate.sh three-js-visualization
```

## ワークフロー作成ガイドライン

新しいワークフローを作成する場合：

1. **タスクの分割可能性を確認**
   - 3つ以上のサブタスクに分割できるか
   - 各サブタスクが5分以上かかるか
   - 並行処理可能か

2. **ワークフローファイルを作成**
   - ファイル名: `タスク名.md`
   - 各エージェントのタスクを明記
   - 依存関係を記載

3. **テンプレート**
   ```markdown
   # ワークフロー名

   ## 概要
   このワークフローの目的

   ## タスク分割

   ### エージェント1: タスク名
   - 具体的なタスク内容
   - 必要なファイル
   - 成果物

   ### エージェント2: タスク名
   - 具体的なタスク内容
   - 必要なファイル
   - 成果物

   ### エージェント3: タスク名
   - 具体的なタスク内容
   - 必要なファイル
   - 成果物

   ## 依存関係
   エージェント間の依存関係があれば記載

   ## 期待される成果
   最終的な成果物
   ```
